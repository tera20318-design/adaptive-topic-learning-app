# v1 To v2 Differences

## Reused Ideas

- Final-report-first orientation
- Source traceability
- Atomic claim tracking
- Explicit release status

These ideas were rewritten in a new module layout rather than imported from `research-os/`.

## Discarded From The Core

- Single master prompt as the primary control surface
- Topic-specific required section headings
- Topic-specific note artifacts in the release gate
- Artifact-count bias
- Implicit mixing of claim capture with claim support
- Theme-specific failure maps in core logic

## New In v2

- Explicit intent classification
- Explicit risk tier classification
- Separate source-role strategy by claim kind
- Domain adapter as a first-class artifact
- Absence-claim handling rules
- Honest scoped/full labeling
- Tone control linked to support status
- Fixture-based regression design separated from the core
